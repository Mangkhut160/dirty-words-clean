# 精神内耗终结者 — 对抗鲁棒性测试设计文档

> 状态：已批准 | 日期：2026-05-25

## 1. 目标

验证 tonebarrier SKILL 双层架构（DFA + LLM）的核心价值命题：**当 DFA 精确匹配失败时，LLM 能否正确兜底**。同时对常见绕过手段建立量化检测率基线，作为发布前的质量证明和后续迭代的回归测试。

## 3. 对抗变异类型定义

### 3.1 格式绕过

**定义**：原脏话词不变，通过插入/替换/编码改变其表面形式。

| 子类型 | 输入示例 | 原始词 | DFA 预期 |
|--------|---------|--------|---------|
| 空格分隔 | "你们这服务真 t m d 差劲" | tmd | ✗ 漏 |
| 符号分隔 | "这 f*u*c*k*i*n*g app" | fucking | ✗ 漏 |
| 全角字符 | "你们ｔｍｄ这破产品" | tmd | ✗ 漏 |
| 下划线连接 | "what the f_u_c_k" | fuck | ✗ 漏 |
| 句点分隔 | "s.h.i.t this is bad" | shit | ✗ 漏 |
| 零宽字符 | "tm​d 破玩意" | tmd | ✗ 漏 |

**生成逻辑**：从 400 词词典中选取能被空格/符号拆分的词（≥3 字母的英文词、≥2 字的中文词），对每个词生成 2-3 种格式变体，嵌入投诉模板。

### 3.2 中文谐音变体

**定义**：用同音或近音字替换脏话中的字，保持发音相似但字形不同。

| 输入示例 | 原始词 | 变体类型 | DFA 预期 |
|---------|--------|---------|---------|
| "真是煞笔设计" | 傻逼 | 同音 | ✓ 命中 |
| "你们草拟吗的" | 操你妈 | 同音 | ✗ 漏 |
| "真尼玛坑" | 你妈 | 同音 | ✗ 漏 |
| "特麼恶心" | 他妈 | 同音 | ✗ 漏 |
| "卧嘈这质量" | 我操 | 同音 | ✗ 漏 |
| "沙雕产品" | 傻屌 | 同音 | ✓ 命中 |

**生成逻辑**：基于 `homophone_guide.md` 中的变体映射表 + 扩展谐音字典。对每一对（原始词 → 变体），生成 1-2 个嵌入投诉模板的对抗样本。同时测试 DFA 是否能命中变体（部分谐音词已在词典中）。

### 3.3 数字/字母替换（Leet）

**定义**：用形似的数字或符号替换字母，或反之。

| 输入示例 | 原始词 | DFA 预期 |
|---------|--------|---------|
| "你 sh1t 一样的设计" | shit | ✓ 命中 |
| "什么 2b 功能" | 二逼 | ✓ 命中 |
| "this is g@rb@g3" | garbage | ✗ 漏 |
| "你 @ss 滚" | ass | ✗ 漏 |
| "b!tch 退钱" | bitch | ✗ 漏 |
| "d1ck move" | dick | ✗ 漏 |
| "f*ck this" | fuck | ✓ 命中(F*) |

**生成逻辑**：对英文脏话应用 Leet 映射表（a→@/4, e→3, i→1/!, o→0, s→5/$ 等），确保至少覆盖映射表中每种替换。`sh1t`/`f*ck`/`2b` 等在现有词典中已覆盖的变体直接命中，作为"DFA 能兜底"的对照组。

### 3.4 中英混杂

**定义**：中文拼音和英文词根混合，形成"杂交"脏话。

| 输入示例 | 构成 | DFA 预期 |
|---------|------|---------|
| "操 nimab 的" | 操 + ni + ma + b | ✗ 漏 |
| "真尼玛 b 的烦" | 尼玛 + b | ✗ 漏 |
| "cnm 的破玩意" | cnm + 中文 | ✓ 命中(cnm) |
| "垃圾 f**king app" | 中文 + 英文 | ✗ 漏(f**king) |
| "sb 一样的服务" | sb + 中文 | ✓ 命中(sb) |
| "你妈的 shit" | 中文 + 英文 | ✗ 漏(中文部分可能命中) |

**生成逻辑**：组合中文脏话词根 + 英文脏话词根 + 拼音缩写。重点是 DFA 无法完整匹配、但语义上明确含脏话的混合形式。

### 3.5 中文拼音和谐音混杂

**定义**：中文拼音缩写与谐音字交替使用，或拼音与中文汉字混排。

| 输入示例 | 构成 | DFA 预期 |
|---------|------|---------|
| "tmd 这煞笔产品" | 拼音 + 谐音 | ✓ 命中(tmd/煞笔) |
| "操 ni 马的" | 中文 + 拼音 | ✗ 漏 |
| "wo 操这质量" | 拼音 + 中文 | ✗ 漏 |
| "cao 你妈的垃圾" | 拼音 + 中文 | ✗ 漏 |

**生成逻辑**：选取高命中率的拼音缩写，将其部分替换为全拼/谐音/中文，测试 LLM 对不完整拼音的上下文补全能力。

### 3.6 讽刺语气 + 语义伪装

**定义**：不含任何脏话词，通过反语、夸张、阴阳怪气表达不满。

| 输入示例 | 类型 | 期望级别 |
|---------|------|---------|
| "你们这物流可真快啊 15 天才到 真不愧是极速达👍" | 反语讽刺 | L3 |
| "售后太棒了 等了两周终于有人回我了 感动的哭了" | 阴阳怪气 | L3 |
| "退款速度绝了，比蜗牛还快" | 夸张讽刺 | L3 |
| "你们的客服一定是哈佛毕业的 太专业了" | 反语 | L3 |
| "从来没有见过这么'好用'的app呢" | 引号反语 | L3 |

**生成逻辑**：手写 10-15 个讽刺模板，填充不同投诉场景（物流、售后、退款、产品质量）。不依赖词典——全部靠 LLM 语义识别。

### 3.7 纯英文 DFA 未命中场景

**定义**：DFA 词典对英文脏话覆盖率有限的真实场景。

| 输入示例 | DFA 预期 | 说明 |
|---------|---------|------|
| "this is bloody rubbish" | ✗ 漏 | "bloody" 不在词典 |
| "what a load of bollocks" | ✗ 漏 | "bollocks" 不在词典 |
| "this app is wank" | ✗ 漏 | "wank" 不在词典 |
| "you mug, refund me" | ✗ 漏 | "mug" 不在词典 |
| "sod this product" | ✗ 漏 | "sod" 不在词典 |
| "this is proper dodgy" | ✗ 漏 | "dodgy" 不在词典 |

**生成逻辑**：从英文脏话词库（不在当前 400 词中的）选取英式/澳式俚语脏话，嵌入英文投诉模板。验证 LLM 的跨文化脏话识别能力。

## 4. 评估指标体系

### 4.1 规则自动化指标

| 指标 | 计算公式 | 目标 |
|------|---------|------|
| 对抗检测率 | 情绪级别 ≥ L3 的样本占比（对 L3+ 期望的样本） | ≥ 80% |
| 过滤干净度 | 净化文本中无可检测脏话残留的样本占比 | ≥ 90% |
| 实体保留率 | validator passed=true 的样本占比 | ≥ 90% |
| 格式合规率 | 两段式格式正确的样本占比 | ≥ 95% |
| DFA 对比增益 | (LLM 检测率 - DFA 检测率) 按对抗类型分组 | ≥ +50pp |

### 4.2 LLM Judge 打分维度

每条输出 1-5 分评估四个维度：

| 维度 | 1 分 | 3 分 | 5 分 |
|------|------|------|------|
| 脏话清除 | 脏话原文原样保留 | 部分脏话残留 | 所有脏话完全清除 |
| 语气转化 | 语气仍然尖锐/愤怒 | 部分改善但仍带情绪 | 完全客观中立 |
| 事实保留 | 关键信息全部丢失 | 部分丢失 | 所有订单号/金额/地址完整 |
| 自然度 | 生硬/不通顺 | 基本通顺 | 读起来像人写的 |

### 4.3 按对抗类型分组报告

| 对抗类型 | 样本数 | DFA 检测率 | LLM 检测率 | 增益 | 过滤干净度 | 综合分 |
|---------|--------|-----------|-----------|------|-----------|--------|
| 格式绕过 | ~30 | ~5% | ? | ? | ? | ? |
| 中文谐音 | ~50 | ~40% | ? | ? | ? | ? |
| Leet 替换 | ~20 | ~40% | ? | ? | ? | ? |
| 中英混杂 | ~20 | ~20% | ? | ? | ? | ? |
| 拼音谐音混杂 | ~15 | ~30% | ? | ? | ? | ? |
| 讽刺语气 | ~15 | 0% | ? | ? | ? | ? |
| 英文未命中 | ~15 | 0% | ? | ? | ? | ? |
| 正常对照组 | ~15 | ~5% | — | — | — | ? |

## 5. 正常对照组

从 `app_negative_reviews.json` 中选取 15 条不含脏话的中英文正常投诉，作为误杀率对照。期望：这些样本中 ≥90% 被判定为 L1-L2（不被误判为脏话）。

## 6. 回归测试集成

对抗样本中选 20 条作为回归子集，整合到 `test_pipeline.py` 第 3 层：

```python
# test_pipeline.py 第 3 层：对抗鲁棒性
adversarial_cases = [
    {"id": "adv_space_bypass_01", "input": "...", "expected_level": 4, "expected_dfa_hit": False},
    {"id": "adv_homophone_01", "input": "...", "expected_level": 4, "expected_dfa_hit": False},
    ...
]
# 检查: DFA 漏了但 LLM 应识别
# 注意: 这些是 DFA-only 测试 + 输出格式检查，LLM 判定需要实际运行 /tonebarrier
```

## 8. 预期效果

| 维度 | 保守估计 | 乐观估计 | 说明 |
|------|---------|---------|------|
| 综合 LLM 检测率 | ≥75% | ≥85% | 对所有 L3+ 对抗样本 |
| DFA→LLM 增益 | ≥50pp | ≥70pp | DFA 漏 → LLM 补的核心指标 |
| 正常样本误杀率 | ≤5% | ≤3% | 15 条正常对照组 |
| 格式绕过最高 | ≥80% | ≥90% | 全角/空格是最容易识别的 |
| 讽刺最难 | ≥60% | ≥75% | 语义理解是 LLM 最大挑战 |
| 综合 F1 (LLM层) | ≥0.75 | ≥0.85 | 全量对抗样本 |

## 9. 与现有组件的关系

| 现有组件 | 对抗测试中的角色 |
|---------|----------------|
| `dfa_filter.py` | 标注每条对抗样本 DFA 是否命中，计算 DFA→LLM 增益 |
| `validator.py` | 验证净化后实体保留 |
| `SKILL.md` | 对抗变异类型来源（homophone_guide.md 的变体表 + few-shot） |
| `test_pipeline.py` | 第 3 层集成对抗回归子集 |
| `benchmark/skill_eval.py` | 复用 parse_skill_output / validate_results 逻辑 |
| `benchmark/report.py` | 复用 Markdown 报告生成模式 |
