## 3. 关键发现：脏话清除率的真相

### 问题根因

e2e_validate.py 的 `check_profanity_removal()` 函数对"净化文本"的定义有误：

```
[情绪判断] 客户情绪激烈，含攻击性语言 — 以下为过滤后内容

DFA 检测到 1 处情绪化表达（煞笔），已过滤。  ← 摘要行（给主管看的检测报告）

客户认为产品设计很差。                        ← 真正的净化文本
```

旧验证器把**整个区域**（含摘要行）当作净化文本，对其运行 DFA 扫描，自然会在摘要行中找到"煞笔"等词并报告"未清除"。

### 修复方案

`parse_skill_output()` 新增逻辑：识别并排除 DFA 摘要行（含"检测到"+"过滤"模式的行），仅对真正的客户文本正文检查脏话残留。

### 真实数据

| 检查范围 | 脏话残留数 | 清除率 |
|----------|-----------|--------|
| 含摘要行（旧） | 30/109 | 72.5% |
| 仅正文（新） | 3/109 | **97.2%** |

## 4. 情绪级别优化

### 4.1 SKILL.md 改进

**旧定义**（模糊）：
```
| 3 | 明显愤怒，含激烈用词或讽刺 |
| 4 | 严重愤怒，含辱骂或人身攻击 |
```

**新定义**（精确）：
```
| 3 | 愤怒语气/讽刺/感叹式粗口（卧槽/我靠/damn/bloody），未直接辱骂对方 |
| 4 | 含辱骂性脏话（傻逼/操你妈/fuck/shit/ass 及其所有变体），无论是否针对人 |
```

新增关键区分规则：
- 谐音变体（沙比/沙幣/尼玛/草拟吗）= 原词伪装 → 按原词级别（级别 4）
- 感叹粗口（卧槽/窝草/damn/bloody）= 情绪宣泄非辱骂 → 级别 3
- 纯客观描述即使内容负面 → 级别 1
